# История России: музейная лента времени

Интерактивный музейный таймлайн по истории России. Проект покрывает пять периодов: Древняя Русь, Московское государство, Российская империя, СССР и современная Россия. В каждом периоде есть четыре элемента: событие, человек, явление и идея.

Весь исторический контент хранится в `data/timeline.json`. Сайт, SVG-постер, PDF и PNG создаются автоматически из этих данных.

## Установка

```bash
npm install
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Python-зависимости устанавливаются только в локальное виртуальное окружение `.venv`.

## Интерактивный сайт

```bash
npm run dev
```

После запуска Vite откроет локальный адрес вида `http://localhost:5173/`.

## Загрузка изображений

```bash
python3 scripts/fetch_images.py
```

Скрипт ищет открытые изображения на Wikimedia Commons, сохраняет их в `public/images/`, обновляет фоновые изображения в `data/timeline.json` и записывает атрибуцию в `data/image_sources.json`.

Если подходящее изображение не найдено, проект продолжает использовать локальную нейтральную SVG-панель. Такие случаи перечисляются в поле `missing` файла `data/image_sources.json`.

## Проверка данных

```bash
python3 scripts/validate_timeline.py
```

Проверяется:

- ровно 5 периодов;
- минимум 4 элемента в каждом периоде;
- наличие типов `событие`, `человек`, `явление`, `идея`;
- описания по 50-150 слов;
- хронологический порядок внутри периодов;
- наличие изображений и источников.

После генерации всех файлов можно проверить и результаты экспорта:

```bash
npm run validate
```

## Экспорт SVG/PDF/PNG

```bash
python3 scripts/render_static_svg.py
python3 scripts/export_static.py
```

Результаты создаются в папке `output/`:

- `output/timeline.svg` — редактируемый статический постер;
- `output/timeline.pdf` — основной файл для сдачи;
- `output/timeline.png` — высококачественный предпросмотр.

## Файл для сдачи

Для финальной сдачи используйте `output/timeline.pdf`. В нём сохранены все 5 периодов, 20 элементов, полные описания и хронологический порядок.

## Источники и лицензии

Исторические ссылки указаны в `data/timeline.json` у каждого элемента. Атрибуция изображений сохраняется в `data/image_sources.json`. Проект не использует удалённые изображения напрямую: все визуальные материалы кэшируются локально в `public/images/`.
